# TechHub 项目分工总结与进展规划

## 一、项目概述

**TechHub** 是一个研发团队任务协同管理平台，采用前后端分离架构：
- **后端**：Python Flask + SQLAlchemy ORM + JWT 认证
- **前端**：Vue 3 + Vite + Element Plus + ECharts
- **数据库**：MySQL（开发环境兼容 SQLite）

- RBAC 权限体系：5 级角色（超管/部门负责人/项目经理/普通成员/数据查看员），9 种细粒度权限控制

- 软删除机制：用户禁用（`is_active`）、项目标记删除（`status='deleted'`）
- 权限申请联动：审批标题含 `[权限申请]` 时，通过后自动授予 `data_viewer` 角色

| **任务管理** | ✅ 100% | 任务 CRUD、四状态流转、四级优先级、指派/截止日/评论、看板拖拽排序 |
| **审批中心** | ✅ 100% | 五种审批类型、审批链自动构建、通过/拒绝处理、统计筛选、审批链可视化 |

### 4.2 技术亮点

1. **拖拽看板**：原生 HTML5 Drag & Drop + 乐观更新策略，失败自动回退刷新
2. **审批链引擎**：动态构建多级审批节点，自动推进到下一节点，支持可视化追踪
3. **权限联动**：通过审批系统申请数据中心权限，审批通过后自动授予角色
4. **数据隔离**：所有查询默认过滤当前用户参与的项目，确保数据安全
5. **活动自动记录**：任务/项目/审批/评论操作自动产生 Activity 记录，驱动团队动态

### 4.3 当前存在的问题

| 问题 | 影响 | 优先级 |
|------|------|--------|
| 无单元测试/接口测试 | 回归风险高 | 🔴 高 |
| 附件功能仅有数据结构，无实际上传 | 审批/任务无法上传文件 | 🟡 中 |
| 无 WebSocket 实时通知 | 审批处理、任务指派无法实时推送 | 🟡 中 |
| 部分 API 无分页（如 `my-tasks`、`performance`） | 数据量大时性能下降 | 🟡 中 |
| 前端图表未使用 `vue-echarts` | 代码稍显冗长 | 🟢 低 |
| 无操作日志审计 | 无法追踪管理操作 | 🟢 低 |
